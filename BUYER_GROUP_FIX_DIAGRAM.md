## Key Insights

### 1. Minimum Loading Time (300ms)
- **Why 300ms?** Human perception sweet spot for "responsive"
- **< 100ms**: Feels instant (no loading indicator needed)
- **100-300ms**: Fast but needs indicator (our range)
- **300-1000ms**: Slow but acceptable
- **> 1000ms**: Slow (needs progress indicator)

### 2. Empty Cache Invalidation
- **Problem**: Empty cache could be stale (data synced after cache)
- **Solution**: Never trust empty cache, always fetch fresh
- **Impact**: Slightly more API calls for empty cases, but correct data

### 3. State Update Timing
- **Problem**: React state updates are async (setLoading might not take effect immediately)
- **Solution**: Use time-based tracking (Date.now()) instead of just state
- **Impact**: Guaranteed minimum loading time regardless of state update timing

### 4. User Perception
- **Users prefer**: Predictable loading â†’ smooth transition
- **Users hate**: Flash of content â†’ jarring change
- **Result**: 300ms loading feels faster than 50ms flash + change
